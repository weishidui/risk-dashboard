# 金融交易风险实时监控与分析平台 - MySQL 表结构说明（增强版）

---

## 1. 数据库说明

数据库名称：

```text
risk_control
```

当前数据库共设计五张核心业务表：

| 表名 | 中文名称 | 主要作用 | 字段数 |
| --- | --- | --- | --- |
| `transaction_history` | Kafka 原始交易流水表 | 存储实时接入的原始交易数据（含全部29个埋点字段） | 29 |
| `user_profile` | 用户历史行为画像表 | 持久化保存离线计算得到的用户画像（含24个特征字段） | 24 |
| `risk_alert` | 实时风控告警结果表 | 存储 Spark Streaming 计算后的风险结果，含告警处理工作流 | 18 |
| `counterparty_blacklist` | 收款方风险黑名单表 | 存储高风险收款方账户，支撑实时风控硬阻断 | 12 |
| `trans_chain` | 资金链路追踪表 | 记录多跳转账链路，检测资金回流/洗钱分层 | 11 |

整体数据流向：

```text
┌──────────────────────────────────────────────────────────────┐
│  数据产生端 (交易者模拟系统)                                    │
│    │                                                         │
│    ▼                                                         │
│  Kafka (Topic: trans-event)                                  │
│    │                                                         │
│    ├──→ transaction_history (原始交易入库)                    │
│    │                                                         │
│    └──→ Spark Streaming 实时消费                              │
│           │                                                  │
│           ├── 查 Redis ←── user_profile (画像缓存)            │
│           ├── 查 Redis ←── counterparty_blacklist (黑名单)    │
│           ├── 查 trans_chain (资金链路)                       │
│           │                                                  │
│           └──→ risk_alert (风控结果入库)                       │
│                                                              │
│  离线批处理 (Spark SQL / MR):                                 │
│    HDFS ← Sqoop ← transaction_history                        │
│    Spark SQL → user_profile → Redis (profile:{user_id})      │
│    Spark SQL → counterparty_blacklist → Redis                 │
│    Spark SQL → trans_chain                                    │
└──────────────────────────────────────────────────────────────┘
```

## 2. 原始交易流水表：`transaction_history`（31个字段）

### 2.1 表用途

`transaction_history` 用于保存 Kafka 接入端收到的原始交易流水数据。字段与 Kafka JSON 报文字段完全一致，覆盖**设备环境、交易行为、收款方信息、身份会话**四个维度共29个字段，支撑全部60条风控规则的实时计算。

### 2.2 字段定义

#### 基础交易信息（9个字段）

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `trans_id` | VARCHAR(64) | 非空，唯一索引 | 交易流水号，格式 `TXN{yyMMddHHmmss}{6位随机}` |
| `user_id` | VARCHAR(64) | 非空，普通索引 | 用户标识，关联画像库主键 |
| `amount` | DOUBLE | 非空 | 本次交易金额 |
| `trans_timestamp` | BIGINT | 非空，普通索引 | 交易时间戳，13位毫秒级 |
| `city` | VARCHAR(64) | 非空，普通索引 | 交易发生地城市名 |
| `geo_location` | VARCHAR(64) | 非空 | 地理经纬度，格式如 `116.3,39.9` |
| `device_id` | VARCHAR(64) | 非空，普通索引 | 设备指纹，硬件唯一标识码 |
| `network_type` | VARCHAR(20) | 非空 | 网络类型：WiFi / 4G / 5G / VPN |
| `dev_score` | INT | 非空 | 设备安全分，范围 0-100 |

#### 设备/环境维度（10个字段，覆盖规则 B1-B6, I1-I6）

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `ip_address` | VARCHAR(45) | 非空 | 客户端IP地址（IPv4/IPv6） |
| `os_type` | VARCHAR(20) | 非空 | 操作系统类型：Android / iOS / Web |
| `os_version` | VARCHAR(30) | 非空 | 系统版本号，如 `Android 8.0` |
| `screen_resolution` | VARCHAR(20) | 非空 | 屏幕分辨率，如 `1080×2400` |
| `battery_level` | INT | 可空 | 电池电量 0-100（移动端获取，Web端为NULL） |
| `root_jailbreak` | TINYINT(1) | 非空 | 是否Root/越狱：0=否 1=是 |
| `sim_operator` | VARCHAR(20) | 可空 | SIM卡运营商：移动/联通/电信（Web端为NULL） |
| `user_agent` | VARCHAR(500) | 非空 | 浏览器/设备User-Agent字符串 |
| `dns_server` | VARCHAR(45) | 可空 | DNS服务器地址 |
| `wifi_ssid` | VARCHAR(100) | 可空 | 当前WiFi名称（脱敏后，4G/5G为NULL） |

#### 交易行为维度（6个字段，覆盖规则 C1-C8, G1-G7）

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `trans_type` | VARCHAR(20) | 非空 | 交易类型：同行转账 / 跨行转账 / 对公转账 |
| `pay_channel` | VARCHAR(20) | 非空 | 支付渠道：bank_card(银行卡) / balance(余额) / wechat(微信) / alipay(支付宝) |
| `input_method` | VARCHAR(20) | 非空 | 输入方式：manual(手动) / paste(粘贴) / autofill(自动填充) |
| `click_duration` | BIGINT | 非空 | 从进入转账页到点击提交的停留时长（毫秒） |
| `note` | VARCHAR(200) | 可空 | 转账备注/附言 |
| `page_url` | VARCHAR(200) | 非空 | 操作页面URL/路由（检测HTTP明文和来源域名） |

#### 收款方维度（3个字段，覆盖规则 F1-F8）

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `counterparty_id` | VARCHAR(64) | 非空，普通索引 | 收款方账户ID，关联黑名单和资金链路 |
| `counterparty_name` | VARCHAR(64) | 非空 | 收款方姓名（脱敏，仅保留首尾字符） |
| `counterparty_bank` | VARCHAR(64) | 非空 | 收款方开户行（用于判断对公/个人） |

#### 身份/会话维度（2个字段，覆盖规则 A1-A8, E3-E4）

| 字段名 | 类型 | 约束 | 说明 |

| `login_fail_count` | INT | 非空 | 本次登录成功前的失败次数（0表示首次成功） |

#### 系统字段

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `create_time` | TIMESTAMP | 默认当前时间，普通索引 | 数据入库时间 |

> **说明**：Kafka 报文字段名为 `timestamp`，MySQL 表中使用 `trans_timestamp`，避免与 MySQL 时间类型名称混淆。

## 3. 用户历史行为画像表：`user_profile`（24个字段）

### 3.1 表用途

`user_profile` 用于持久化保存离线计算得到的用户历史行为画像（24个字段）。Redis 中的 `profile:{user_id}` 用于实时风控快速查询，MySQL 中的 `user_profile` 作为画像数据的持久化备份。画像由 **Spark SQL 离线批处理** 每天凌晨定时计算更新。

### 3.2 字段定义

#### 行为基线

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `user_id` | VARCHAR(64) | 非空，唯一索引 | 用户标识 |
| `avg_amt_30d` | DOUBLE | 非空 | 过去30天单笔平均金额 |
| `common_cities` | VARCHAR(255) | 非空 | 过去3个月高频城市 Top 5，逗号分隔 |
| `common_devs` | VARCHAR(255) | 非空 | 用户常用设备 ID 列表，逗号分隔 |
| `common_pay_channels` | VARCHAR(100) | 非空 | 常用支付渠道，逗号分隔 |
| `common_trans_types` | VARCHAR(100) | 非空 | 常用交易类型，逗号分隔 |
| `common_counterparties` | VARCHAR(500) | 非空 | 常用收款方ID列表，逗号分隔 |

#### 历史快照

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `last_trans_ts` | BIGINT | 非空 | 上一笔交易发生的时间戳 |
| `last_city` | VARCHAR(64) | 非空 | 上一笔交易发生城市 |
| `last_ip` | VARCHAR(45) | 非空 | 上次登录IP地址 |
| `last_login_time` | BIGINT | 非空 | 上次登录时间戳 |

#### 账户信息

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `registration_time` | BIGINT | 非空 | 账户注册时间戳 |
| `total_balance` | DOUBLE | 非空 | 账户当前余额 |
| `single_limit` | DOUBLE | 非空 | 单笔转账限额 |
| `daily_limit` | DOUBLE | 非空 | 日累计转账限额 |
| `monthly_limit` | DOUBLE | 非空 | 月累计转账限额 |
| `account_status` | VARCHAR(20) | 非空 | normal(正常) / frozen(冻结) / flagged(标记) / dormant(休眠) |

#### 累计统计（离线计算，实时更新）

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `login_count_24h` | INT | 非空 | 近24小时登录次数 |
| `trans_count_24h` | INT | 非空 | 近24小时转账笔数 |
| `trans_amount_24h` | DOUBLE | 非空 | 近24小时转账总金额 |
| `trans_count_7d` | INT | 非空 | 近7天转账笔数 |
| `cancel_retry_count` | INT | 非空 | 取消后重试次数（同一金额/收款方） |

#### 风险标记

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `risk_tags` | VARCHAR(255) | 可空 | 风险标签：high_risk_area / money_mule / victim，逗号分隔 |
| `risk_score` | INT | 非空 | 用户综合风险评分（-100到100，负值越负越低风险） |

#### 系统字段

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `update_time` | TIMESTAMP | 自动更新时间，普通索引 | 画像更新时间 |

## 4. 实时风控告警结果表：`risk_alert`20个字段）

### 4.1 表用途

`risk_alert` 用于保存 Spark Streaming 实时消费 Kafka 数据、查询 Redis 用户画像/收款方黑名单并完成60条风险规则判断后的结果。SpringBoot 后端和 Vue 大屏主要查询该表进行风险监控展示及告警处理。

### 4.2 字段定义

#### 告警基本信息

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `alert_id` | VARCHAR(64) | 非空，唯一索引 | 告警编号，格式 `ALT{yyyyMMddHHmmss}{6位随机}` |
| `trans_id` | VARCHAR(64) | 非空，普通索引 | 交易流水号，关联原始交易 |
| `user_id` | VARCHAR(64) | 非空，普通索引 | 用户标识，关联用户画像 |
| `amount` | DOUBLE | 非空 | 本次交易金额 |
| `city` | VARCHAR(64) | 非空 | 交易城市 |

#### 风险判定结果

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `hit_rules` | VARCHAR(500) | 非空 | 命中的风险规则标签，多个用分号分隔，如 `"A1账户盗用;C1金额突变;D2异地瞬移"` |
| `final_score` | INT | 非空 | 综合风险评分，范围 0-300+ |
| `risk_level` | VARCHAR(20) | 非空，普通索引 | 风险等级：低危 / 中危 / 高危 / 极度危险 |
| `alert_loc` | VARCHAR(64) | 非空 | 告警位置，用于大屏地图展示 |

#### 告警处理工作流（新增）

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `status` | VARCHAR(20) | 非空，普通索引 | 处理状态：pending(待处理) / processing(处理中) / verified(已核验) / blocked(已拦截) / closed(已关闭) |
| `handler` | VARCHAR(64) | 可空 | 处理人标识 |
| `handle_time` | BIGINT | 可空 | 处理时间戳（毫秒） |
| `handle_remark` | VARCHAR(500) | 可空 | 处理备注/意见 |

#### 风控详情（新增，便于前端展示和分析）

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `counterparty_id` | VARCHAR(64) | 可空 | 关联收款方ID |
| `ip_address` | VARCHAR(45) | 可空 | 交易IP地址 |
| `is_new_device` | TINYINT(1) | 非空 | 是否新设备：0=否 1=是 |
| `is_new_counterparty` | TINYINT(1) | 非空 | 是否首次转账给该收款方：0=否 1=是 |
| `chain_id` | VARCHAR(64) | 可空 | 关联资金链路ID |

#### 系统字段

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `raw_json` | TEXT | 可空 | Kafka 原始 JSON 报文 |
| `create_time` | TIMESTAMP | 默认当前时间，普通索引 | 告警入库时间 |

### 4.3 风险等级规则（5级）

| 综合风险评分 | 风险等级 | 系统动作 | 处理含义 |
| --- | --- | --- | --- |
| 0 - 40 | 低危 | 自动放行 | 交易正常完成 |
| 41 - 70 | 中危 | 要求二次验证 | 弹出人脸/短信验证 |
| 71 - 120 | 高危 | 人工审核 + 资金挂起 | 等待风控分析师审核 |
| > 120 | 极度危险 | 自动拦截 + 冻结账户 | 直接拒绝，账户冻结 |
| 命中硬阻断 | 直接拦截 | 立即拦截（不计算总分） | 收款方黑名单等极端规则 |

### 4.4 告警处理工作流

```
pending(待处理)
    │
    ▼
processing(处理中)  ──→ verified(已核验，放行)
    │
    └──→ blocked(已拦截，确认拦截)
    │
    ▼
closed(已关闭)
```

每次状态变更记录 `handler`（操作人）+ `handle_time`（操作时间）+ `handle_remark`（处理意见）。

## 5. 收款方风险黑名单表：`counterparty_blacklist`（12个字段）🆕

### 5.1 表用途

`counterparty_blacklist` 用于存储高风险收款方账户信息，由**离线 Spark SQL 定时计算**或**管理员手动标记**维护。实时风控引擎查询此表，命中的交易按风险等级处理（high=硬阻断、medium=+50分）。

### 5.2 字段定义

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `counterparty_id` | VARCHAR(64) | 非空，唯一索引 | 收款方账户ID |
| `counterparty_name` | VARCHAR(64) | 可空 | 收款方姓名（脱敏） |
| `risk_level` | VARCHAR(20) | 非空，普通索引 | 风险等级：high(黑名单，硬阻断) / medium(灰名单，+50分) / low(观察名单) |
| `risk_type` | VARCHAR(100) | 非空 | 风险类型：fraud(诈骗) / money_laundering(洗钱) / phishing(钓鱼) / gambling(赌博) / other |
| `source` | VARCHAR(50) | 非空 | 来源：police_report(公安通报) / internal_detection(内部判定) / user_report(用户举报) / external_blacklist(外部黑名单) |
| `total_received_24h` | DOUBLE | 非空 | 近24小时收款总额（离线计算） |
| `total_received_7d` | DOUBLE | 非空 | 近7天收款总额（离线计算） |
| `unique_payers_24h` | INT | 非空 | 近24小时付款方数量（检测多对一特征） |
| `registration_days` | INT | 非空 | 账户注册天数（≤7天为新注册） |
| `risk_tags` | VARCHAR(255) | 可空 | 补充风险标签，逗号分隔 |
| `created_time` | TIMESTAMP | 默认当前时间 | 记录创建时间 |
| `update_time` | TIMESTAMP | 自动更新时间 | 记录更新时间 |

### 5.3 自动入库规则（Spark SQL 定时任务）

| 条件 | 自动标记 | 风险等级 |
|------|---------|---------|
| `unique_payers_24h ≥ 20` 且 `total_received_24h > 500000` | 多对一集中收款 | medium → high |
| `total_received_24h / total_received_7d > 0.8` 且 `unique_payers_24h ≥ 10` | 快进快出 | medium |
| `registration_days ≤ 7` 且 `total_received_24h > 100000` | 新注册大额收款 | medium |

### 5.4 Redis 对应结构

```text
counterparty:{counterparty_id}
```

```bash
HSET counterparty:C_FRAUD_001 \
  risk_level "high" \
  risk_type "fraud" \
  total_received_24h 850000 \
  unique_payers_24h 25
```

## 6. 资金链路追踪表：`trans_chain`（11个字段）🆕

### 6.1 表用途

`trans_chain` 用于记录多跳转账链路，由**Spark Streaming 实时写入**（每笔交易记录一跳），由**Spark SQL 离线分析**（检测环形回流和多级跳转）。支撑规则 H1-H6 的计算。

### 6.2 字段定义

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `chain_id` | VARCHAR(64) | 非空，普通索引 | 链路批次ID（同一资金链路的所有跳共享） |
| `trans_id` | VARCHAR(64) | 非空，普通索引 | 交易流水号 |
| `from_user_id` | VARCHAR(64) | 非空，普通索引 | 转出方用户ID |
| `to_user_id` | VARCHAR(64) | 非空，普通索引 | 转入方用户ID |
| `hop_order` | INT | 非空 | 第几跳（1/2/3/...） |
| `amount` | DOUBLE | 非空 | 该跳金额 |
| `trans_time` | BIGINT | 非空 | 交易时间戳（毫秒） |
| `chain_depth` | INT | 非空 | 当前链路总深度（总跳数） |
| `is_loop` | TINYINT(1) | 非空 | 是否形成环：0=否 1=是（如 A→B→A） |
| `create_time` | TIMESTAMP | 默认当前时间 | 记录创建时间 |

### 6.3 链路构建规则

```
示例：A→B→C→A 环形链路

chain_id: CHAIN_001
┌────┬──────────┬─────────┬───────┬───────┬────────┬─────────────┬─────────────┬─────────┐
│ id │ chain_id │ trans_id│ from  │ to    │ hop    │ amount      │ chain_depth │ is_loop │
├────┼──────────┼─────────┼───────┼───────┼───────┼─────────────┼─────────────┼─────────┤
│ 1  │ CHAIN_001│ TXN_01  │ A     │ B     │ 1     │ 50000       │ 1           │ 0       │
│ 2  │ CHAIN_001│ TXN_02  │ B     │ C     │ 2     │ 48000       │ 2           │ 0       │
│ 3  │ CHAIN_001│ TXN_03  │ C     │ A     │ 3     │ 47000       │ 3           │ 1 ← 环 │
└────┴──────────┴─────────┴───────┴───────┴───────┴─────────────┴─────────────┴─────────┘
当 hop_order=3 且 to_user_id 出现在之前的 from_user_id 中 → is_loop=1 → H1+70分 H5+65分
```

### 6.4 链路追踪实现

`chain_id` 生成规则（在 Spark Streaming 中实现）：

```python
# 伪代码
if from_user_id 在最近30分钟内有转入记录:
    复用已有的 chain_id，hop_order += 1
elif to_user_id 在最近30分钟内有转出记录:
    复用已有的 chain_id，hop_order += 1
else:
    生成新的 chain_id，hop_order = 1

# 环检测
if to_user_id 出现在当前链路的 from_user_id 列表中:
    is_loop = 1
```

## 7. 五张表之间的关系

### 7.1 核心关联

```text
transaction_history.trans_id
        │
        ▼
risk_alert.trans_id           ← 一笔交易 → 一条告警（可能）
```

```text
user_profile.user_id
        │
        ▼
risk_alert.user_id            ← 风控时按 user_id 查画像
        │
        ▼
Redis profile:{user_id}       ← 画像缓存
```

```text
transaction_history.counterparty_id
        │
        ├──→ counterparty_blacklist.counterparty_id   ← 收款方黑名单检查
        │
        └──→ trans_chain (from/to)                    ← 资金链路构建
```

### 7.2 关系汇总

| 关系 | 源表.字段 | 目标表.字段 | 说明 |
| --- | --- | --- | --- |
| 交易→告警 | `transaction_history.trans_id` | `risk_alert.trans_id` | 一条原始交易经风控计算后产生一条告警 |
| 画像→告警 | `user_profile.user_id` | `risk_alert.user_id` | 风控计算时根据用户ID查询画像生成结果 |
| 画像↔Redis | `user_profile.user_id` | `profile:{user_id}` | MySQL持久化画像，Redis提供实时查询缓存 |
| 收款方→黑名单 | `transaction_history.counterparty_id` | `counterparty_blacklist.counterparty_id` | 收款方风险查询 |
| 交易→链路 | `transaction_history.trans_id` | `trans_chain.trans_id` | 每笔交易记录一跳 |
| 黑名单↔Redis | `counterparty_blacklist.counterparty_id` | `counterparty:{id}` | MySQL持久化黑名单，Redis缓存加速查询 |
