# 金融交易风险实时监控平台 — 数据库连接指南

## 1. 数据库概览

| 项目 | 值 |
|------|-----|
| 数据库名称 | `risk_control` |
| 数据库类型 | MySQL 5.7+ / H2 (开发环境) |
| 字符集 | utf8mb4 |
| 连接池 | Druid (阿里巴巴) |

### 三张核心业务表

| 表名 | 用途 | 数据来源 |
|------|------|----------|
| `transaction_history` | Kafka 原始交易流水 | Kafka → Spark Streaming |
| `user_profile` | 用户历史行为画像 | HDFS/Spark 离线计算 |
| `risk_alert` | 实时风控告警结果 | Spark Streaming 实时风控引擎 |

### 数据流向

```
数据产生端 → Kafka → transaction_history
HDFS/Spark 离线计算 → user_profile → Redis
Kafka + Redis → Spark Streaming → risk_alert
```

## 2. 环境说明

项目支持两套环境，通过 `spring.profiles.active` 切换：

| 环境 | 数据库 | Redis | 适用场景 |
|------|--------|-------|----------|
| **dev** (默认) | H2 内存数据库 | 可选（降级处理） | 本地开发、调试 |
| **prod** | MySQL | 必须 | 生产部署 |

## 3. 开发环境 (dev) — 零配置启动

无需安装任何外部依赖，直接启动即可：

```bash
cd risk-dashboard-backend

# Maven 启动
mvn spring-boot:run

# 或先打包再启动
mvn clean package -DskipTests
java -jar target/risk-dashboard-backend-1.0.0.jar
```

### dev 环境特点

- 使用 **H2 内存数据库**，数据库名 `risk_control`，完全兼容 MySQL 语法
- 启动时自动执行 `src/main/resources/db/schema.sql` 建表 + 插入模拟数据
- H2 控制台可访问 `http://localhost:8080/h2-console`（JDBC URL 填 `jdbc:h2:mem:risk_control`）
- Redis 不可用时自动降级，不影响系统启动

## 4. 生产环境 (prod) — 接入 MySQL

### 4.1 安装 MySQL

```bash
# Ubuntu/Debian
sudo apt install mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全检查
sudo mysql_secure_installation
```

### 4.2 创建数据库

登录 MySQL 后执行：

```sql
CREATE DATABASE IF NOT EXISTS risk_control
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

-- 创建应用用户 (可选，替换 your_password)
CREATE USER IF NOT EXISTS 'risk_admin'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON risk_control.* TO 'risk_admin'@'localhost';
FLUSH PRIVILEGES;
```

### 4.3 执行建表脚本

```bash
mysql -u root -p risk_control < src/main/resources/db/schema.sql
```

或者去掉模拟数据部分，仅执行建表语句（去掉文件末尾 `INSERT INTO risk_alert ...` 部分）。

### 4.4 修改连接配置

编辑 `src/main/resources/application-prod.yml`：

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://你的MySQL IP:3306/risk_control?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: risk_admin        # 改为你的用户名
    password: ${MYSQL_PASSWORD:你的密码}   # 或设置环境变量 MYSQL_PASSWORD
```

### 4.5 启动应用

```bash
# 方式一：Maven
mvn spring-boot:run -Dspring-boot.run.profiles=prod

# 方式二：Jar 包
java -jar target/risk-dashboard-backend-1.0.0.jar --spring.profiles.active=prod

# 方式三：环境变量
export SPRING_PROFILES_ACTIVE=prod
java -jar target/risk-dashboard-backend-1.0.0.jar
```

## 5. 三张核心表结构速查

### 5.1 transaction_history（交易流水表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | 自增主键 |
| `trans_id` | VARCHAR(64) UNIQUE | 交易流水号 |
| `user_id` | VARCHAR(64) INDEX | 用户标识 |
| `amount` | DOUBLE | 交易金额 |
| `trans_timestamp` | BIGINT INDEX | 13位毫秒时间戳 |
| `city` | VARCHAR(64) INDEX | 交易城市 |
| `geo_location` | VARCHAR(64) | 经纬度 (如 `116.3,39.9`) |
| `device_id` | VARCHAR(64) | 设备指纹 |
| `network_type` | VARCHAR(20) | 网络类型 (WiFi/4G/5G/VPN) |
| `dev_score` | INT | 设备安全分 (0-100) |
| `create_time` | TIMESTAMP | 入库时间 (自动) |

### 5.2 user_profile（用户画像表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | 自增主键 |
| `user_id` | VARCHAR(64) UNIQUE | 用户标识 |
| `avg_amt_30d` | DOUBLE | 近30天平均金额 |
| `common_cities` | VARCHAR(255) | 高频城市 (逗号分隔) |
| `common_devs` | VARCHAR(255) | 常用设备 (逗号分隔) |
| `last_trans_ts` | BIGINT | 上一笔交易时间戳 |
| `last_city` | VARCHAR(64) | 上一笔交易城市 |
| `update_time` | TIMESTAMP | 更新时间 (自动) |

### 5.3 risk_alert（风控告警表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | 自增主键 |
| `alert_id` | VARCHAR(64) | 告警编号 |
| `trans_id` | VARCHAR(64) INDEX | 关联交易流水号 |
| `user_id` | VARCHAR(64) INDEX | 用户标识 |
| `amount` | DOUBLE | 交易金额 |
| `city` | VARCHAR(64) | 交易城市 |
| `hit_rules` | VARCHAR(255) | 命中规则标签 (分号分隔) |
| `final_score` | INT | 综合风险评分 (0-100) |
| `risk_level` | VARCHAR(20) INDEX | 风险等级 (低危/中危/高危) |
| `alert_loc` | VARCHAR(64) | 告警位置 (用于地图展示) |
| `raw_json` | TEXT | Kafka 原始 JSON 报文 |
| `create_time` | TIMESTAMP INDEX | 告警入库时间 (自动) |

### 风险等级与处理动作映射

| 评分 | 等级 | 处理动作 |
|------|------|----------|
| 0-59 | 低危 | PASS (放行) |
| 60-80 | 中危 | VERIFY (核验) |
| 81-100 | 高危 | BLOCK (拦截) |

## 6. 表关系图

```
transaction_history.trans_id ──────→ risk_alert.trans_id
user_profile.user_id ──────────────→ risk_alert.user_id
user_profile.user_id ─── Redis ────→ profile:{user_id}
```

- 一条原始交易 (`transaction_history`) 经过风控计算后产生一条风险告警 (`risk_alert`)
- 风控计算时根据 `user_id` 查询用户画像 (`user_profile` / Redis `profile:{user_id}`)
- MySQL 保存画像快照，Redis 提供实时查询缓存

## 8. 常见问题

**Q: 开发环境数据在哪？重启后数据会丢失吗？**
A: dev 模式使用 H2 内存数据库，每次重启数据都会丢失并从 schema.sql 重新初始化。适合开发和调试。

**Q: 如何查看 Druid 监控？**
A: prod 模式下访问 `http://localhost:8080/druid/`，默认账号 `admin`，密码 `admin`（可通过 `DRUID_PASSWORD` 环境变量修改）。

**Q: MySQL 连接报错 "Public Key Retrieval is not allowed"？**
A: jdbcUrl 末尾已加 `allowPublicKeyRetrieval=true`。如果还报错，检查 MySQL 用户认证插件：
```sql
ALTER USER 'risk_admin'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_password';
```
