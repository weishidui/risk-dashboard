# 金融交易风险实时监控与分析平台 - 需求分析文档（增强版）

---

## 1. 引言

### 1.1 项目背景
针对日益复杂的金融欺诈手段（盗刷、洗钱、电信诈骗、羊毛党等），本项目旨在利用大数据分布式计算技术，构建一套"Lambda 架构"的风控系统。系统覆盖**转账全流程**的50种交易风险，兼顾历史数据的深度挖掘（离线画像）与当前交易的即时判定（实时风控），实现对高风险交易的秒级拦截与可视化监控。

### 1.2 核心路径归属说明

| 路径名称 | 核心组件 | 对应需求模块 | 业务职能 |
| :--- | :--- | :--- | :--- |
| **实时流处理路** | Kafka + **Spark Streaming** | **模块三：实时风控引擎** | 对每一笔新交易执行60条风险规则评分，秒级判定并报警 |
| **离线批处理路** | **HDFS** + Spark SQL/MR | **模块二：历史画像分析** | 处理海量历史数据，计算用户行为基准、收款方风险库、资金链路 |

### 1.3 新增模块：交易者操作端

| 模块 | 技术选型 | 业务职能 |
| :--- | :--- | :--- |
| **模块五：交易者模拟系统** | SpringBoot + Vue + Kafka | 模拟真实用户转账操作，产生带完整埋点数据的交易报文，发送至 Kafka |

### 1.4 风控覆盖全景

本项目覆盖转账全流程共 **50种风险**，按转账前/中/后三大阶段细分为9个子类，通过60条可编码的评分规则实现全覆盖。

## 2. 系统功能模块需求

### 2.1 模块一：全链路数据采集 (基础支撑)

* **交易者操作端埋点**：在交易者前端/APP埋点采集20个新增字段，覆盖设备环境、操作行为、收款方信息、会话信息四个维度。
* **流量中转(Nginx)**：负责接收采集端请求并进行负载均衡，保证接入层高可用。
* **缓冲队列(Kafka)**：作为分布式消息中间件，支撑高并发流量接入，Topic: `trans-event`。

### 2.2 模块二：离线行为画像模块 (基于 HDFS/批处理路径)

* **历史数据同步**：定期通过 Sqoop 将业务库数据同步至 **HDFS**，建立 ODS/DWD/DWS 数据分层。
* **深度特征计算**：利用 Spark SQL 处理 HDFS 海量数据，计算用户行为基准（30天均值、常用城市/设备/收款方、日/月累计指标）。
* **收款方风险库**：构建 `counterparty_blacklist` 表，统计收款方资金特征（多对一集中转入、快进快出）。
* **资金链路追踪**：构建 `trans_chain` 表，检测环形回流和多级跳转。
* **画像下发**：将计算结果导出至 **Redis** 高速缓存（`profile:{user_id}` / `counterparty:{counterparty_id}`），支撑实时风控引擎的极速关联查询。

### 2.3 模块三：实时风控引擎模块 (基于 Spark Streaming/流计算路径)

* **实时流解析**：Spark Streaming 实时读取 Kafka Topic `trans-event`，实现微批次准实时计算。
* **多维度关联查询**：查询 Redis 获取用户画像、设备历史、收款方风险信息。
* **60条规则并行判定**：将当前交易特征与画像基线进行比对，执行累加评分。
* **硬阻断机制**：收款方黑名单等极端规则命中后直接拦截，无需计算总分。
* **结果实时入库**：风险判定结果秒级写入 MySQL `risk_alert` 表，供前端展示。
* **告警处理工作流**：支持待处理→处理中→已核验/已拦截→关闭的完整流程。

### 2.4 模块四：可视化看板模块 (数据展示)

* **实时监控**：基于 Vue+ECharts 实现风险大屏，6个核心指标卡片 + 趋势图 + 风险等级饼图 + 规则分类饼图 + 城市柱状图 + 风险速度表 + 最新告警列表。
* **交易流水监控**：瀑布流展示交易数据，VPN/低分设备/大额交易高亮标记。
* **风险告警管理**：分页查询 + 按等级/规则/状态筛选 + 标记处理操作（含处理人、处理意见）。
* **风险地理分布**：ECharts 中国地图 + 城市散点(涟漪效果) + 城市排行。
* **数据统计分析**：多维度饼图 + 24小时趋势对比线。
* **系统配置**：数据接入接口文档 + 风险评分规则配置表（管理员可动态调整阈值）。

### 2.5 模块五：交易者模拟系统 (数据产生端)

* **用户登录/注册**：模拟账户体系，支持创建不同画像类型用户（普通/VIP/新用户/僵尸号）。
* **账户管理**：余额查看、交易限额管理、常用信息维护。
* **转账操作**：同行转账、跨行转账、对公转账，含完整表单（金额、收款方、城市选择、设备模拟）。
* **安全校验流程**：交易密码 + 人脸识别(模拟) + 短信验证码(模拟)。
* **预设测试场景**：一键触发盗刷/洗钱/账户被盗/羊毛党等场景，验证风控系统响应。
* **交易记录查看**：展示历史交易及风控判定结果（成功/被拦截/待审核）。

## 4. 风险评分规则体系

### 4.0 评分模型设计原则

| 原则 | 说明 |
|------|------|
| **累加制** | 每条命中的规则独立加分，多规则命中 = 累积高分 |
| **权重分级** | 高危特征权重高(35-80分)，辅助特征权重低(10-35分) |
| **硬阻断** | 收款方黑名单(high级别)等极端规则，命中直接拦截，不计算总分 |
| **时效衰减** | 同一用户同一规则60秒内重复命中，第二次只加50%分值 |

### 4.1 综合评分阈值

| 总分区间 | 风险等级 | 系统动作 | 用户侧表现 |
|---------|---------|---------|-----------|
| **0 - 40** | 🟢 低危 | 自动放行 | 转账正常完成 |
| **41 - 70** | 🟡 中危 | 要求二次验证（人脸/短信） | 弹出验证窗口 |
| **71 - 120** | 🟠 高危 | 人工审核 + 资金挂起 | 交易挂起，等待审核 |
| **> 120** | 🔴 极度危险 | 自动拦截 + 冻结账户 | 直接拒绝 |
| **命中硬阻断** | 🔴 直接拦截 | 任何一条硬阻断规则命中即拦截 | 直接拒绝 |

### 4.2 A类：账户安全规则（8条）

| # | 规则名称 | 判定条件（可编码逻辑） | 分值 | 类型 |
|---|---------|----------------------|------|------|
| A1 | **账户盗用** | `login_fail_count ≥ 5` 且 `ip_address ≠ last_ip` 且 `amount > 5000` | +60 | 硬规则 |
| A2 | **撞库攻击** | 同IP 5分钟内 `login_fail_count ≥ 10` 个不同 `user_id`（全局统计） | +50 | 硬规则 |
| A3 | **新设备登录转账** | `device_id ∉ common_devs` 且 `trans_type = 'transfer'` 且 `amount > 5000` | +30 | 软规则 |
| A4 | **休眠账户唤醒** | `(now - last_trans_ts) > 180天` 且 `amount > avg_amt_30d × 5` | +40 | 软规则 |
| A5 | **新注册大额转账** | `(now - registration_time) < 24小时` 且 `amount > 10000` | +50 | 硬规则 |
| A6 | **账户被标记** | `account_status = 'flagged'` 或 `risk_tags` 含 `'victim'` | +35 | 软规则 |
| A7 | **多账号同设备** | 同一 `device_id` 关联 ≥ 5 个不同 `user_id`（全局统计） | +45 | 硬规则 |
| A8 | **异常快速重登录** | `(login_time - last_login_time) < 60秒` 且 `ip_address ≠ last_ip` | +25 | 软规则 |

### 4.4 C类：金额特征规则（8条）

| # | 规则名称 | 判定条件（可编码逻辑） | 分值 | 类型 |
|---|---------|----------------------|------|------|

| C2 | **大额整数转账** | `amount ≥ 50000` 且 `amount % 10000 = 0` | +15 | 软规则 |

| C5 | **小额测试后大额** | `amount < 1` 且该用户此前有 `amount < 1` 的测试交易记录 | +10 | 软规则 |

| C8 | **月累计超限** | `(trans_amount_7d × 4.3 + amount) > monthly_limit` | +20 | 软规则 |

## 6. 系统角色与安全性需求

### 6.1 用户角色定义

| 角色 | 权限 | 人数 |
|------|------|------|
| **普通用户（交易者）** | 登录、查看余额、发起转账、查看交易记录 | 不限 |
| **风控分析师** | 查看仪表盘大屏、告警管理（标记处理）、交易流水查询、风险地图 | 2-3人 |
| **系统管理员** | 全部权限 + 规则阈值配置 + 用户管理 + 收款方黑名单维护 | 1-2人 |
| **审计员** | 只读查看 + 报表导出（Excel/PDF） | 1人 |

### 6.2 数据安全与隐私

* **隐私脱敏**：展示层对姓名、手机号进行星号脱敏（`138****8888`）、收款方姓名仅显示首尾字符。
* **权限分级**：离线HDFS原始数据目录仅限计算账号访问，禁止普通用户导出。
* **操作审计**：所有告警处理操作记录 `handler` + `handle_time` + `handle_remark`。
* **传输加密**：所有对外API强制HTTPS（`page_url`检测）。内部集群间通信走内网IP。
