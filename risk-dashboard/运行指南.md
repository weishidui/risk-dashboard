## 第一步：启动后端服务

```bash

cd /home/wsd/shixun/risk-dashboard-backend

# 2. Maven 编译打包（跳过测试，加快速度）
```

## 第四步：页面功能操作指南

### 4.1 实时监控仪表盘（首页）

**访问**：http://localhost:8081/#/dashboard

**页面组成**（从上到下）：

| 区域 | 内容 | 说明 |
|------|------|------|
| 第一行 | 6 个指标卡片 | 秒级交易量、放行数、核验数、拦截数、活跃用户、平均延迟 |
| 第二行左 | 趋势折线图 | 点击切换：交易量 / 告警量 / 拦截率 三种趋势 |
| 第二行右 | 风险等级饼图 | 高/中/低危占比 |
| 第三行左 | 规则类型饼图 | 金额异常/地理偏离/异地瞬移/环境风险占比 |
| 第三行中 | 城市柱状图 | 各城市交易活跃度 Top10 |
| 第三行右 | 风险速度表 | 实时平均风险指数（0-100 仪表盘） |
| 第四行 | 最新告警列表 | 表格展示最近告警，含评分标签、状态 |

**操作**：
- 趋势图左上角 **切换按钮**：查看不同类型的趋势
- 右上角 **手动刷新按钮**：立即拉取最新数据
- 页面 **每 5 秒自动刷新**

---

### 4.2 交易流水监控

**访问**：http://localhost:8081/#/transaction

**功能**：实时查看所有交易流水，类似"瀑布流"效果。

**重点关注**：
- 网络类型显示 **红色 VPN 标签** 的交易（环境风险）
- 设备安全分 **低于 50 的橙色/红色标签**（设备异常）
- 大额交易（> ¥10,000）红色高亮显示

---

### 4.3 风险告警管理

**访问**：http://localhost:8081/#/alerts

**筛选操作**：
1. 下拉选择 **风险等级**：高危 / 中危 / 低危
2. 下拉选择 **处理动作**：放行 / 核验 / 拦截
3. 点击 **查询按钮**

**处理告警**：
1. 在表格右侧找到"待处理"的告警
2. 点击 **标记处理** 按钮
3. 确认对话框中选择"确定"
4. 状态变为绿色 **已处理**

---

### 4.4 风险地理分布

**访问**：http://localhost:8081/#/risk-map

**地图操作**：
- **鼠标拖拽**：平移地图
- **鼠标滚轮**：缩放地图
- **红色圆点**：高危告警城市（圆越大告警越多）
- **涟漪动画**：告警数 > 3 的城市有红色扩散特效

**右侧面板**：
- 城市风险排行表
- 最新高危告警详情滚动列表

---

### 4.5 数据统计分析

**访问**：http://localhost:8081/#/analysis

**三个图表**：
- 风险等级分布饼图（带颜色区分）
- 触发规则类型饼图
- 24小时三维趋势对比（高危/中危/低危三条线）

## 常见问题

**Q: 启动报 "Address already in use"？**
```bash
# 查看占用端口的进程
lsof -i:8080
# 强制终止
kill -9 <PID>
```

**Q: 前端页面空白？**
- 确认后端已启动：`curl http://localhost:8080/api/dashboard/health`
- 打开浏览器控制台查看报错（F12 → Console）

**Q: Swagger 页面打不开？**
- 确认已添加 `spring.mvc.pathmatch.matching-strategy=ant_path_matcher`
- 访问地址必须是 `/swagger-ui/index.html`（不是 `/swagger-ui.html`）

**Q: 如何重置演示数据？**
- 开发版使用 H2 内存库，**重启后端即可重置**
- 生产版请执行：`DELETE FROM t_alert; DELETE FROM t_transaction;`

```
lsof -ti:8080 | xargs kill -9 2>/dev/null; lsof -ti:8081 | xargs kill -9 2>/dev/null; sleep 1; echo "端口已释放" && lsof -ti:8080 2>/dev/null || echo "8080 空闲" && lsof -ti:8081 2>/dev/null || echo "8081 空闲"
```
