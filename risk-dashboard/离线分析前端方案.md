# 离线分析前端可视化方案

> 基于 [离线分析结果清单](./离线分析结果清单.md)，当前离线 MapReduce 已产出 6 层 HDFS 数据 + MySQL/RDS 结果。本文档说明前端需要新增哪些页面来可视化这些结果。

---

## 一、离线分析已产出什么（后端数据已就绪）

```
HDFS ODS (原始)  ──→  HDFS DWD (清洗 600,000 条)  ──→  HDFS DWS (4 张汇总表)  ──→  HDFS ADS (49,410 画像)
                                                                                      │
                                                                                      ▼
                                                                MySQL user_profile (49,410 行)
                                                                Redis profile:*       (49,410 key)
                                                                Redis device_risk:*   (97,670 key)
                                                                Redis counterparty:*  (15 key)
                                                                MySQL counterparty_blacklist (3,337 行)
                                                                MySQL trans_chain           (5,155 行)
```

## 三、需要新增的 4 个页面

### 3.1 离线总览看板（优先级：⭐⭐⭐）

**路由：** `/offline-overview`

**数据来源：** MySQL 聚合查询 + Redis `INFO keyspace`

**展示内容：**

```
┌─────────────────────────────────────────────────────────┐
│  离线分析总览                                dt=20260706 │
├────────────┬────────────┬────────────┬──────────────────┤
│ DWD 清洗   │ 用户画像    │ 高风险用户  │ 设备风险画像      │
│ 600,000    │ 49,410     │ 7,024      │ 97,670           │
├────────────┴────────────┴────────────┴──────────────────┤
│                                                         │
│  ┌──────────────────────┐  ┌──────────────────────────┐ │
│  │ 用户风险分分布(柱状图) │  │ 账户状态分布(饼图)        │ │
│  │ risk_score 分段统计   │  │ normal/frozen/flagged/   │ │
│  │                      │  │ dormant                  │ │
│  └──────────────────────┘  └──────────────────────────┘ │
│                                                         │
│  ┌──────────────────────────────────────────────────┐   │
│  │ 离线任务状态 (6 个分区)                            │   │
│  │ dwd_trans_event           ✅ _SUCCESS            │   │
│  │ dws_user_transaction_daily ✅ _SUCCESS            │   │

│  │ dws_device_daily          ✅ _SUCCESS            │   │
│  │ dws_user_risk_daily       ✅ _SUCCESS            │   │
│  │ ads_user_profile          ✅ _SUCCESS            │   │
│  └──────────────────────────────────────────────────┘   │
│                                                         │
│  ┌──────────────────────┐  ┌──────────────────────────┐ │
│  │ 收款方风险分布         │  │ 链路深度分布              │ │
│  │ 黑名单: 3,337         │  │ 环形链路: (from MySQL)   │ │

│  └──────────────────────┘  └──────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**后端接口需求：**

| 接口 | 说明 |
|------|------|
| `GET /api/offline/overview` | 返回所有汇总数字（DWD行数、画像数、高风险数、设备数、黑名单数、链路数） |
| `GET /api/offline/task-status` | 返回 6 个分区的 `_SUCCESS` 状态 |
| `GET /api/offline/risk-distribution` | 用户风险评分分段统计（0-20, 21-40, 41-60, 61-80, 81-100, 100+） |

│  ┌──────────────────────────────────────────────────┐   │
│  │ 收款方风险等级分布 (饼图)                           │   │
│  │ high / medium / low                               │   │
│  └──────────────────────────────────────────────────┘   │
│                                                         │

**后端接口需求：**

| 接口 | 说明 |
|------|------|
| `GET /api/counterparty-risk/stats` | 收款方风险总览（总数、黑名单数、高风险缓存数） |
| `GET /api/counterparty-risk/list` | 分页查询收款方风险列表 |
| `GET /api/counterparty-risk/{counterpartyId}` | 查询收款方风险详情 |

> 注：如果 `dws_counterparty_daily` 的 47,042 行数据尚未导出到 MySQL，需要先通过 Sqoop 导出到 MySQL 表 `dws_counterparty_daily`，或后端直接读取 HDFS 文件。

## 六、实施建议

### 按优先级分两批做：

**第一批（核心、数据已就绪、工作量小）：**

| 任务 | 工作量 |
|------|:---:|
| 后端：离线总览 3 个接口 + 高风险用户 2 个接口 | 1 Controller + 1 Service |

| 任务 | 工作量 |

| 前端：设备风险页面 `DeviceRisk.vue` | 1 页面 |
