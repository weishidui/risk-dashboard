## 一、概述

本项目是**金融交易风险实时监控与分析平台**，采用Lambda架构，同时覆盖**实时流处理**（秒级风控拦截）和**离线批处理**（T+1画像计算、收款方黑名单、资金链路分析）。

全链路覆盖50种交易风险、60条评分规则、29个埋点字段、5张数据库表、4层数据仓库分层。

## 四、环节 2：数据采集

### 4.1 交易端 → Kafka（实时写入）

```
交易者操作端 (端口8082)
    │
    │ POST /api/trader/transfer
    │ JSON: 29个字段
    ▼
trader-backend
    │
    │ KafkaProducer.send("trans-event", json)
    ▼
Kafka Broker (mrzk:9092)
    │ Topic: trans-event
    │ Partition: 3, Replication: 1
    │
    ├──→ ① 实时流: Spark Streaming 消费 (环节6)
    │
    ├──→ ② 落盘: Flume → HDFS (本节)
    │
    └──→ ③ 简单写入: Kafka Consumer → transaction_history 表
```

### 4.2 Kafka → HDFS 实时落盘（关键路径）

**为什么需要这条路径？** 实时数据除了供Spark Streaming消费外，必须同步落盘到HDFS，作为离线计算的数据源（ODS层）。

**方案**：使用 **Flume** 实现 Kafka → HDFS 的数据同步。

```
Flume Agent 配置 (mrzk节点):

Source:   Kafka Source
          topic = trans-event
          groupId = flume-hdfs-group

Channel:  Memory Channel (缓冲)

Sink:     HDFS Sink
          路径: /user/hive/warehouse/risk_control.db/ods_trans_event/dt=%Y%m%d/
          格式: TEXT (一行一个JSON)
          滚动策略: 每128MB或每10分钟生成一个新文件
```

### 4.3 数据采集要点

1. **Kafka为缓冲中心**：交易端只需要写Kafka，不关心下游有多少消费者。
2. **Flume负责Kafka→HDFS**：配置简单，不需要写代码。
3. **消息格式统一为JSON**：29个字段名固定，下游按名解析。
4. **每条消息带唯一trans_id**：用于去重和问题追溯。

## 五、环节 3：数据清洗（MapReduce：ODS → DWD）

> **HDFS ODS层的原始JSON，经过MapReduce清洗后写入DWD层。**

### 5.1 清洗流程

```
ODS层 (HDFS)
  /ods_trans_event/dt=20260703/
  ├── FlumeData.xxx.json  (一行一个JSON，可能重复/脏数据)
  │
  ▼  MapReduce Job 1: ODS→DWD清洗
  │
DWD层 (HDFS)
  /dwd_trans_event/dt=20260703/
  └── part-r-00000 (Parquet格式，清洗后数据)
```

### 5.2 MapReduce 代码结构（需编写4个类）

#### TransCleanMapper.java

```java
/**
 * Mapper: 读取ODS层的JSON行，解析并清洗
 * 输入: (行偏移量, 一行JSON字符串)
 * 输出: (trans_id, DwdTransWritable对象)
 */
public class TransCleanMapper extends Mapper<LongWritable, Text, Text, DwdTransWritable> {

    @Override
    protected void map(LongWritable key, Text value, Context context) {
        String line = value.toString();

        // 1. 解析JSON
        JSONObject json = JSON.parseObject(line);

        // 2. 校验 — 过滤脏数据
        String user_id = json.getString("user_id");
        Double amount = json.getDouble("amount");
        Long timestamp = json.getLong("timestamp");
        if (user_id == null || user_id.isEmpty()) return;  // user_id为空，丢弃
        if (amount == null || amount <= 0) return;          // 金额非法，丢弃
        if (timestamp == null || timestamp <= 0) return;    // 时间戳非法，丢弃

        // 3. 标准化
        String city = normalizeCity(json.getString("city"));       // "北京市" → "北京"
        String payChannel = normalizePayChannel(json.getString("pay_channel"));
        String networkType = json.getString("network_type");
        String note = json.getString("note");
        if (note == null) note = "";
        Long clickDuration = json.getLong("click_duration");
        if (clickDuration == null) clickDuration = 0L;

        // 4. 构建DWD对象
        DwdTransWritable dwd = new DwdTransWritable();
        dwd.setTransId(json.getString("trans_id"));
        dwd.setUserId(user_id);
        dwd.setAmount(amount);
        dwd.setTransTimestamp(timestamp);
        dwd.setCity(city);
        dwd.setGeoLocation(json.getString("geo_location"));
        dwd.setDeviceId(json.getString("device_id"));
        dwd.setNetworkType(networkType);
        dwd.setDevScore(json.getIntValue("dev_score"));
        // ... 设置所有29个字段 ...
        dwd.setProcessTime(System.currentTimeMillis());  // 新增：ETL处理时间

        context.write(new Text(dwd.getTransId()), dwd);
    }

    private String normalizeCity(String city) {
        if (city == null) return "未知";
        if (city.equals("北京市")) return "北京";
        if (city.equals("上海市")) return "上海";
        return city;
    }

    private String normalizePayChannel(String ch) {
        if (ch == null) return "unknown";
        switch (ch) {
            case "银行卡": return "bank_card";
            case "微信": return "wechat";
            case "支付宝": return "alipay";
            case "余额": return "balance";
            default: return ch;
        }
    }
}
```

#### TransCleanReducer.java

```java
/**
 * Reducer: 按 trans_id 去重，只保留第一条
 * 输入: (trans_id, [DwdTransWritable列表])
 * 输出: (NullWritable, DwdTransWritable)
 */
public class TransCleanReducer extends Reducer<Text, DwdTransWritable, NullWritable, DwdTransWritable> {

    @Override
    protected void reduce(Text key, Iterable<DwdTransWritable> values, Context context) {
        // 去重：取第一条
        for (DwdTransWritable val : values) {
            context.write(NullWritable.get(), val);
            break;  // 只输出第一条
        }
    }
}
```

#### TransCleanDriver.java

```java
/**
 * ODS→DWD清洗任务提交
 */
public class TransCleanDriver {
    public static void main(String[] args) throws Exception {
        Configuration conf = new Configuration();
        Job job = Job.getInstance(conf, "ODS-to-DWD Trans Clean");

        job.setJarByClass(TransCleanDriver.class);
        job.setMapperClass(TransCleanMapper.class);
        job.setReducerClass(TransCleanReducer.class);

        job.setMapOutputKeyClass(Text.class);
        job.setMapOutputValueClass(DwdTransWritable.class);
        job.setOutputKeyClass(NullWritable.class);
        job.setOutputValueClass(DwdTransWritable.class);

        // 输入: ODS路径
        FileInputFormat.addInputPath(job, new Path(args[0]));  // hdfs://.../ods_trans_event/dt=20260703/
        // 输出: DWD路径
        FileOutputFormat.setOutputPath(job, new Path(args[1])); // hdfs://.../dwd_trans_event/dt=20260703/

        System.exit(job.waitForCompletion(true) ? 0 : 1);
    }
```

## 九、完整案例：从"点击消除"到大屏上的统计图

> **用一个真实场景串起全链路**。

### 场景
> 老板要求大屏上显示**"过去 1 小时玩家消除次数"**，每 3 秒刷新。

### 全链路拆解

```
步骤 1【数据产生】   玩家在 H5 客户端点击消除
                    客户端埋点: tt.track('elim', {...})
                            ↓
步骤 2【数据采集】   HTTP POST → 服务端 → Kafka topic: event_elim
                            ↓
步骤 3【数据清洗】   Flink 消费 Kafka
                    - 解析 JSON
                    - 过滤 player_id 为空的
                    - 标准化时间戳
                            ↓
步骤 4【数据存储】   落 ClickHouse:
                    CREATE TABLE dwd_event_elim (...)
                            ↓
步骤 5【数据中台】   聚合到 DWS:
                    CREATE MATERIALIZED VIEW dws_elim_last_hour
                    AS SELECT count() FROM dwd_event_elim
                       WHERE ts > now() - INTERVAL 1 HOUR
                            ↓
步骤 6【数据应用】   大屏后端每 3s 查询:
                    SELECT count FROM dws_elim_last_hour
                    推送到前端 → ECharts 数字滚动
```

### 入门级简化版（用 MySQL 实现）

```sql
-- 1) 数据产生 + 采集：直接写 MySQL（够用就行）
INSERT INTO t_event_log (event_code, player_id, payload, create_time)
VALUES ('elim', 10001, '{"combo":3}', NOW());

-- 2) 清洗：定时任务每分钟跑
INSERT INTO dws_elim_last_hour
SELECT
  DATE_FORMAT(NOW(), '%Y-%m-%d %H:%i') AS dt,
  COUNT(*) AS elim_count
FROM t_event_log
WHERE event_code = 'elim'
  AND create_time > DATE_SUB(NOW(), INTERVAL 1 HOUR)
ON DUPLICATE KEY UPDATE elim_count = VALUES(elim_count);

-- 3) 应用：大屏 API
SELECT elim_count FROM dws_elim_last_hour ORDER BY dt DESC LIMIT 1;
```

> **即使不用 Kafka/Flink，MySQL + 定时任务也能跑起来**！这就是入门思路。
