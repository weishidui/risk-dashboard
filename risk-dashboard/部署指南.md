# 金融交易风险实时监控平台 — 部署指南

## 1. 集群拓扑

根据需求分析文档(1) 第 6.2 节，5 节点集群规划如下：

| 节点 | IP | 组件 | 角色 |
|------|-----|------|------|
| **mrzk** | 192.168.140.104 | Nginx, Zookeeper, Kafka, Redis, MySQL | 公共中间件节点 |
| **master0** | 192.168.140.100 | NameNode, ResourceManager, Spark Master, Zookeeper | 集群主控节点 |
| **slave1** | 192.168.140.101 | DataNode, NodeManager, Spark Worker, Zookeeper | 核心计算节点 1 |
| **slave2** | 192.168.140.102 | DataNode, NodeManager, Spark Worker | 核心计算节点 2 |
| **slave3** | 192.168.140.103 | DataNode, NodeManager, **SpringBoot, Vue** | **计算+应用展示节点** |

## 2. 部署目标

| 组件 | 目标节点 | IP | 端口 |
|------|---------|-----|------|
| risk-dashboard-backend (SpringBoot) | **slave3** | 192.168.140.103 | 8080 |
| risk-dashboard-frontend (Vue) | **slave3** | 192.168.140.103 | 80 (Nginx) 或 8081 (开发) |

## 3. 依赖中间件

| 中间件 | 所在节点 | IP | 端口 |
|------|---------|-----|------|
| MySQL | mrzk | 192.168.140.104 | 3306 |
| Redis | mrzk | 192.168.140.104 | 6379 |
| Kafka | mrzk | 192.168.140.104 | 9092 |

## 4. 部署步骤

### 4.1 后端部署 (slave3)

```bash
# 1. 编译打包
cd /home/wsd/shixun/risk-dashboard-backend
/tmp/apache-maven-3.9.6/bin/mvn clean package -DskipTests

# 2. 上传到 slave3
scp target/risk-dashboard-backend-1.0.0.jar root@192.168.140.103:/opt/risk-dashboard/

# 3. 在 slave3 上启动
ssh root@192.168.140.103
cd /opt/risk-dashboard
nohup java -jar risk-dashboard-backend-1.0.0.jar > app.log 2>&1 &

# 4. 验证
curl http://192.168.140.103:8080/api/dashboard/overview
```

### 4.2 前端部署 (slave3)

```bash
# 1. 构建生产包
cd /home/wsd/shixun/risk-dashboard-frontend
npm run build

# 2. 上传到 slave3
scp -r dist/ root@192.168.140.103:/opt/risk-dashboard/frontend/

# 3. 配置 Nginx 或直接使用开发服务器
# 方式 A: Nginx (推荐)
#   server { listen 80; root /opt/risk-dashboard/frontend; ... }
#
# 方式 B: 开发模式
ssh root@192.168.140.103
cd /opt/risk-dashboard/frontend
nohup npx serve -s . -l 8081 > frontend.log 2>&1 &
```

### 4.3 后端配置要点

`application.yml` 中确保以下配置指向 mrzk 节点：

```yaml
spring:
  datasource:
    url: jdbc:mysql://192.168.140.104:3306/risk_db
  redis:
    host: 192.168.140.104
    port: 6379
```
