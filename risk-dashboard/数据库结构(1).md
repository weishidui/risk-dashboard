# 金融交易风险实时监控与分析平台 - MySQL 表结构说明

## 1. 数据库说明

数据库名称：

```text
risk_control
```

当前数据库共设计三张核心业务表：

| 表名 | 中文名称 | 主要作用 |
| --- | --- | --- |
| `transaction_history` | Kafka 原始交易流水表 | 存储实时接入的原始交易数据 |
| `user_profile` | 用户历史行为画像表 | 持久化保存离线计算得到的用户画像 |
| `risk_alert` | 实时风控告警结果表 | 存储 Spark Streaming 计算后的风险结果 |

整体数据流向：

```text
数据产生端 -> Kafka -> transaction_history
HDFS/Spark 离线计算 -> user_profile -> Redis
Kafka + Redis -> Spark Streaming -> risk_alert
```

---

## 2. 原始交易流水表：transaction_history

### 2.1 表用途

`transaction_history` 用于保存 Kafka 接入端收到的原始交易流水数据。该表字段与 Kafka JSON 报文字段基本保持一致，便于后续数据追踪、问题排查、离线分析和数据回放。

### 2.2 字段定义

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `trans_id` | VARCHAR(64) | 非空，唯一索引 | 交易流水号，全局唯一 ID |
| `user_id` | VARCHAR(64) | 非空，普通索引 | 用户标识，关联画像库主键 |
| `amount` | DOUBLE | 非空 | 本次交易金额 |
| `trans_timestamp` | BIGINT | 非空，普通索引 | 交易时间戳，13 位毫秒级时间戳 |
| `city` | VARCHAR(64) | 非空，普通索引 | 交易发生地城市名 |
| `geo_location` | VARCHAR(64) | 非空 | 地理经纬度，格式如 `116.3,39.9` |
| `device_id` | VARCHAR(64) | 非空 | 设备指纹，硬件唯一标识码 |
| `network_type` | VARCHAR(20) | 非空 | 网络类型：WiFi、4G、5G、VPN |
| `dev_score` | INT | 非空 | 设备安全分，范围 0-100 |
| `create_time` | TIMESTAMP | 默认当前时间，普通索引 | 数据入库时间 |

说明：Kafka 报文字段名为 `timestamp`，MySQL 表中使用 `trans_timestamp`，避免与 MySQL 时间类型名称混淆。

## 3. 用户历史行为画像表：user_profile

### 3.1 表用途

`user_profile` 用于持久化保存离线计算得到的用户历史行为画像。Redis 中的 `profile:{user_id}` 主要用于实时风控快速查询，而 MySQL 中的 `user_profile` 作为画像数据的持久化备份和展示数据源。

### 3.2 字段定义

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `user_id` | VARCHAR(64) | 非空，唯一索引 | 用户标识 |
| `avg_amt_30d` | DOUBLE | 非空 | 过去 30 天单笔平均金额 |
| `common_cities` | VARCHAR(255) | 非空 | 过去 3 个月高频城市 Top 3，逗号分隔 |
| `common_devs` | VARCHAR(255) | 非空 | 用户常用设备 ID 列表，逗号分隔 |
| `last_trans_ts` | BIGINT | 非空 | 上一笔交易发生的毫秒级时间戳 |
| `last_city` | VARCHAR(64) | 非空 | 上一笔交易发生城市 |
| `update_time` | TIMESTAMP | 自动更新时间，普通索引 | 画像更新时间 |

### 3.3 Redis 对应结构

Redis Key：

```text
profile:{user_id}
```

示例：

```bash
HSET profile:U1001 \
  avg_amt_30d 1800.00 \
  common_cities "北京,上海,杭州" \
  common_devs "D9001,D9002" \
  last_trans_ts 1782790000000 \
  last_city "北京"
```

## 4. 实时风控告警结果表：risk_alert

### 4.1 表用途

`risk_alert` 用于保存 Spark Streaming 实时消费 Kafka 数据、查询 Redis 用户画像并完成风险规则判断后的结果。SpringBoot 后端和 Vue 大屏主要查询该表进行风险监控展示。

### 4.2 字段定义

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | BIGINT | 主键，自增 | 自增主键 |
| `alert_id` | VARCHAR(64) | 非空 | 告警编号 |
| `trans_id` | VARCHAR(64) | 非空，普通索引 | 交易流水号，关联原始交易 |
| `user_id` | VARCHAR(64) | 非空，普通索引 | 用户标识 |
| `amount` | DOUBLE | 非空 | 本次交易金额 |
| `city` | VARCHAR(64) | 非空 | 交易城市 |
| `hit_rules` | VARCHAR(255) | 非空 | 命中的风险规则标签，多个规则用分号分隔 |
| `final_score` | INT | 非空 | 综合风险评分，范围 0-100 |
| `risk_level` | VARCHAR(20) | 非空，普通索引 | 风险等级：低危、中危、高危 |
| `alert_loc` | VARCHAR(64) | 非空 | 告警位置，用于大屏地图展示 |
| `raw_json` | TEXT | 可空 | Kafka 原始 JSON 报文 |
| `create_time` | TIMESTAMP | 默认当前时间，普通索引 | 告警入库时间 |

### 4.3 风险等级规则

| 综合风险评分 | 风险等级 | 处理含义 |
| --- | --- | --- |
| 0-59 | 低危 | 放行 |
| 60-80 | 中危 | 需要核验 |
| 81-100 | 高危 | 建议拦截 |

## 5. 三张表之间的关系

```text
transaction_history.trans_id
        |
        v
risk_alert.trans_id
```

```text
user_profile.user_id
        |
        v
risk_alert.user_id
```

关系说明：

| 关系 | 说明 |
| --- | --- |
| `transaction_history.trans_id` -> `risk_alert.trans_id` | 一条原始交易经过风控计算后，可能产生一条风险告警 |
| `user_profile.user_id` -> `risk_alert.user_id` | 风控计算时根据用户 ID 查询画像，再生成风险结果 |
| `user_profile` -> Redis `profile:{user_id}` | MySQL 保存画像快照，Redis 提供实时查询缓存 |

## 6. 当前数据库表清单

| 表名 | 存储阶段 | 主要消费者 |
| --- | --- | --- |
| `transaction_history` | 原始接入层 | 数据回放、离线分析、问题排查 |
| `user_profile` | 离线画像层 | Redis 同步任务、后台查询 |
| `risk_alert` | 实时结果层 | SpringBoot API、Vue 风控大屏 |
