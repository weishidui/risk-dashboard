# 金融交易风险实时监控与分析平台 - 需求分析文档

## 1. 引言
### 1.1 项目背景
针对日益复杂的金融欺诈手段，本项目旨在利用大数据分布式计算技术，构建一套“Lambda 架构”的风控系统。系统需兼顾历史数据的深度挖掘（离线画像）与当前交易的即时判定（实时风控），实现对高风险交易的秒级拦截与可视化监控。

### 1.2 核心路径归属说明
| 路径名称 | 核心组件 | 对应需求模块 | 业务职能 |
| :--- | :--- | :--- | :--- |
| **实时流处理路** | Kafka + **Spark Streaming** | **模块三：实时风控引擎** | 负责对每一笔新进来的交易进行秒级检测，匹配风险规则并报警。 |
| **离线批处理路** | **HDFS** + Spark SQL/MR | **模块二：历史画像分析** | 负责处理海量历史存量数据，计算用户的长期行为习惯（如近1个月的平均消费）。 |

---

## 2. 系统功能模块需求

### 2.1 模块一：全链路数据采集 (基础支撑)
*   **模拟产生逻辑**：由脚本模拟终端产生 JSON 格式交易日志，包含设备环境、地理位置、交易详情。
*   **流量中转(Nginx)**：负责接收采集端请求并进行负载均衡，保证接入层高可用。
*   **缓冲队列(Kafka)**：作为分布式消息中间件，支撑高并发流量接入，为下游提供解耦的数据流。

### 2.2 模块二：离线行为画像模块 (基于 HDFS/批处理路径)
*   **历史数据同步**：定期通过 Sqoop 将业务库数据同步至 **HDFS**，建立 ODS/DWD/DWS 数据分层。
*   **深度特征计算**：利用 Spark SQL 处理 HDFS 海量数据，挖掘用户行为基准（如：活跃时段、常驻城市、消费限额）。
*   **画像下发**：将计算结果导出至 **Redis** 高速缓存，支撑实时流的极速关联查询。

### 2.3 模块三：实时风控引擎模块 (基于 Spark Streaming/流计算路径)
*   **实时流解析**：Spark Streaming 实时读取 Kafka 消息，实现微批次准实时计算。
*   **规则实时判定**：将当前交易特征与 Redis 中的画像基准进行比对，执行异地检测、金额突变检测及频次核验。
*   **结果实时入库**：风险判定结果秒级写入 MySQL 结果表，供前端展示。

### 2.4 模块四：可视化看板模块 (数据展示)
*   **实时监控**：基于 Vue+Echarts 实现风险大屏，瀑布流展示交易数据，地图红点闪烁提示风险发生地。
*   **风险分类统计**：饼图展示不同风险类型（盗刷、洗钱、环境异常）的占比分布。

## 3. 数据字典规范

### 3.1 实时交易流水报文（Kafka 接入端）
| 字段名称 | 字段代码 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| 交易流水号 | `trans_id` | String | 全局唯一 ID |
| 用户标识 | `user_id` | String | 关联画像库的主键 |
| 交易金额 | `amount` | Double | 本次交易金额 |
| 交易时间戳 | `timestamp` | Long | 13位毫秒级时间戳 |
| 交易城市 | `city` | String | 交易发生地城市名 |
| 地理经纬度 | `geo_location` | String | 坐标 (如 "116.3, 39.9") |
| 设备指纹 | `device_id` | String | 硬件唯一标识码 |
| 网络类型 | `network_type` | String | WiFi/4G/5G/VPN |
| 设备安全分 | `dev_score` | Int | 0-100（模拟环境安全度） |

### 3.2 用户历史行为画像（HDFS 计算 -> Redis 存储）
| 字段名称 | 字段代码 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| 日均消费金额 | `avg_amt_30d` | Double | 过去30天单笔平均金额 |
| 常用城市列表 | `common_cities` | List | 过去3个月高频城市 Top 3 |
| 常用设备列表 | `common_devs` | List | 用户习惯使用的设备 ID 列表 |
| 上次交易时间 | `last_trans_ts` | Long | 上一笔交易发生的精确时间 |
| 上次交易城市 | `last_city` | String | 上一笔交易发生的城市 |

### 3.3 实时风控预警结果（MySQL 输出端）
| 字段名称 | 字段代码 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| 预警编号 | `alert_id` | String | 唯一告警 ID |
| 触发规则标签 | `hit_rules` | String | 如 "金额突变; 异地瞬移" |
| 综合风险评分 | `final_score` | Int | 0-100 分（评分越高越危险） |
| 风险等级 | `risk_level` | String | 高危/中危/低危 |
| 地理位置信息 | `alert_loc` | String | 发生告警的城市 |

## 4. 业务场景逻辑与评分体系

### 4.1 典型业务流程
1.  **离线路**：HDFS 定期通过历史数据更新 Redis 中的 `avg_amt_30d` 与 `last_city`。
2.  **实时路**：Spark Streaming 读取当前交易，查询 Redis 获取画像，代入下方评分规则进行累加计算。

### 4.2 风险评分规则逻辑
| 规则名称 | 判定条件 (逻辑描述) | 风险分值 |
| :--- | :--- | :--- |
| **金额异常** | `amount > avg_amt_30d * 3` (当前消费超历史均值3倍) | +30分 |
| **地理偏离** | `city` 不在 `common_cities` 列表中 | +20分 |
| **异地瞬移** | (当前坐标 - `last_city` 坐标) / 时间间隔 > 1000km/h | +80分 |
| **环境风险** | `network_type == 'VPN'` 或 `dev_score < 50` | +40分 |

*   **综合处理动作**：总分 < 60（绿色/放行）；60-80（黄色/核验）；> 80（红色/拦截）。

## 5. 系统角色与安全性需求

### 5.1 用户角色定义
*   **普通用户**：行为触发者。系统需在不影响其支付体验（延迟 < 2秒）的前提下完成监控。
*   **风控分析师**：使用大屏系统监控风险动向，并根据告警结果进行业务复核。
*   **系统管理员**：负责维护集群稳定性及动态调整风险判定阈值。

### 5.2 数据安全与隐私
*   **隐私脱敏**：在展示层对敏感信息（如身份证、手机号）进行星号脱敏（例：138****8888）。
*   **权限分级**：离线 HDFS 原始数据目录仅限计算账号访问，禁止普通用户导出。

## 6. 非功能需求与运行环境

### 6.1 性能指标
*   **实时性**：从 Kafka 接收数据到写入风险表，全链路处理延迟 < 1 秒。
*   **吞吐量**：支持单秒千量级并发交易的实时比对计算。
