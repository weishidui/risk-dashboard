## 一、架构定位

本项目属于 Lambda 架构中的 **Serving Layer（数据服务与展示层）**，负责接收上游计算系统的处理结果，完成数据持久化、缓存加速、实时推送和可视化展示。

```
                        ┌──────────────────────────┐
                        │   当前代码仓库实现范围      │
                        │                          │
Kafka ──→ Spark Streaming ──→ REST API ──→ MySQL + Redis ──→ Vue 大屏
                │                   │            │               │
           (不在本仓库)         数据接入层    双写缓存层      前端展示层
                                  │            │               │
HDFS ──→ Spark SQL/MR ───────────┘            │               │
                │                              │               │
           (不在本仓库)                   └────────┬──────────┘
                                                   │
                                            实时推送(WebSocket)
                                            告警处理工作流
                                            统计分析聚合
```

### 2.2 各功能详细说明

#### 2.2.1 风险等级评分体系

[RiskLevelEnum.java](risk-dashboard-backend/src/main/java/com/finance/risk/dashboard/common/RiskLevelEnum.java) 严格对应需求文档 4.1 节，提供了 `fromScore(int)` 方法自动判定风险等级：

| 分值区间 | 等级 | 动作 | 系统行为 |
|----------|------|:---:|----------|
| 0 - 40 | 低危 | PASS | 自动放行 |
| 41 - 70 | 中危 | VERIFY | 要求二次验证 |
| 71 - 120 | 高危 | BLOCK | 人工审核 + 资金挂起 |
| > 120 | 极度危险 | BLOCK | 自动拦截 + 冻结账户 |
| 硬阻断 | 直接拦截 | BLOCK | 命中即拦截 |

#### 2.2.2 WebSocket 实时推送

[RiskWebSocketHandler.java](risk-dashboard-backend/src/main/java/com/finance/risk/dashboard/websocket/RiskWebSocketHandler.java) 支持 3 个推送频道，向前端大屏实时广播：

| 频道 Topic | 推送内容 | 前端消费 |
|------------|----------|----------|
| `/topic/alert` | 最新告警数据 JSON | Dashboard.vue 实时告警推送流 |
| `/topic/transaction` | 实时交易数据 JSON | TransactionFlow.vue 瀑布流 |
| `/topic/metrics` | 实时指标更新 JSON | Dashboard.vue 指标卡片刷新 |

#### 2.2.3 仪表盘数据聚合

[MetricsServiceImpl.getDashboardData()](risk-dashboard-backend/src/main/java/com/finance/risk/dashboard/service/impl/MetricsServiceImpl.java) 一次性返回仪表盘首页全部数据：

- 6 个核心指标：总交易量 / 放行数 / 核验数 / 拦截数 / 平均风险评分 / 系统延迟
- 3 条趋势线：交易量趋势 / 告警量趋势 / 拦截率趋势（近 24 小时）
- 3 组分布数据：风险等级分布 / 规则类型分布 / 城市活跃度 Top10
- 地理告警点位：高危城市经纬度坐标
- 最新 10 条告警

#### 2.2.4 告警处理工作流

完整的告警生命周期管理，对应需求文档 2.3 节：

```
                   PUT /api/alert/{alertId}/status

pending ──受理──→ processing ──放行──→ verified ──关闭──→ closed
                      │        拦截                    ↑
                      └────────→ blocked ──────────────→┘
```

```
Spark SQL 离线计算结果
        │
        ▼ POST /api/data/profile  (24个画像字段)
        │
ProfileServiceImpl
        │
        ├──→ MySQL user_profile 表 (持久化)
        │     ├── avg_amt_30d        近30天平均金额
        │     ├── common_cities      常用城市列表 Top5
        │     ├── common_devs        常用设备列表
        │     ├── common_pay_channels 常用支付渠道
        │     ├── common_counterparties 常用收款方
        │     ├── last_trans_ts      上次交易时间
        │     ├── last_city          上次交易城市
        │     ├── last_ip            上次登录IP
        │     ├── last_login_time    上次登录时间
        │     ├── registration_time  注册时间
        │     ├── total_balance      账户余额
        │     ├── single_limit       单笔限额
        │     ├── daily_limit        日累计限额
        │     ├── monthly_limit      月累计限额
        │     ├── account_status     账户状态
        │     ├── login_count_24h    近24h登录次数
        │     ├── trans_count_24h    近24h转账笔数
        │     ├── trans_amount_24h   近24h转账总额
        │     ├── trans_count_7d     近7天转账笔数
        │     ├── cancel_retry_count 取消重试次数
        │     ├── risk_tags          风险标签
        │     ├── risk_score         用户风险评分(-100~100)
        │     └── update_time        画像更新时间
        │
        └──→ Redis: risk:profile:{userId}  (24H TTL)

实时风控引擎查询: redisTemplate.opsForValue().get("risk:profile:" + userId)
```

## 五、总结

### 实时分析：数据接入 + 展示链完整

从 API 接收 → MySQL+Redis 双写 → WebSocket 推送 → 仪表盘大屏，**整个消费链路完整实现**。缺失的是上游 Spark Streaming 规则引擎的计算代码。

### 离线分析：后端 API 就绪，前端缺口大

- 后端为画像、黑名单、资金链路都提供了数据接收和查询 API
- 用户画像的 MySQL + Redis 双写存储完整实现
- 但**前端只有 DataAnalysis.vue 一个统计图表页**，黑名单管理、资金链路追踪、用户画像浏览这三个页面均未开发

### 建议优先补充的前端页面

1. **收款方黑名单管理页** — 风险等级筛选 + 分页表格 + 详情抽屉，后端 API 已完整就绪
2. **资金链路追踪页** — 链路列表 + 环形回流高亮 + 深度分布图，后端 API 已完整就绪
3. **用户画像浏览页** — 用户列表 + 画像详情弹窗 + Redis 缓存状态，需补充查询 API
