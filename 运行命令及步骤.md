# 风险监控系统运行步骤

## 1. 源码目录

| 模块 | 本快照目录 | 集群运行目录 |
|---|---|---|
| 离线分析 | `offline-risk-mapreduce` | `master0:/home/master0/javacode/risk-profile-mapreduce` |
| 实时风控 | `realtime-risk-streaming` | `master0:/home/master0/risk-streaming` |
| Dashboard 前后端 | `risk-dashboard` | `slave3:/home/master0/risk-dashboard` |
| Hadoop JobHistory 配置 | `cluster-config/mapred-site.xml` | 各 Hadoop 节点的 `/home/master0/hadoop-2.7.6/etc/hadoop/mapred-site.xml` |

集群地址：`master0=192.168.125.100`，`slave3=192.168.125.113`，`mrzk=192.168.125.104`。

## 2. 启动基础服务

在 `master0` 执行：

```bash
~/start-all-risk
```

离线启动脚本会自动检查并启动 JobHistory Server。也可手工核验：

```bash
export JAVA_HOME=/home/master0/jdk1.8.0_171
export HADOOP_HOME=/home/master0/hadoop-2.7.6
export PATH=$JAVA_HOME/bin:$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH
jps -l | grep JobHistoryServer
(echo > /dev/tcp/192.168.125.100/10020) >/dev/null 2>&1 && echo HISTORY_READY
```

## 3. 离线分析

先由数据产生端将历史交易写入 MySQL，并确认 `trans_timestamp` 覆盖目标 30 天窗口。`create_time` 是入库时间，离线窗口只依据 `trans_timestamp`。

在 `master0` 执行：

```bash
cd /home/master0/javacode/risk-profile-mapreduce

# 分析 2026-06-13 00:00:00 至 2026-07-13 00:00:00 的交易，不包含 7 月 13 日。
./start_offline.sh 20260713
```

该命令默认持续输出日志；按 `Ctrl+C` 仅退出日志查看，任务继续在后台运行。

```bash
# 后台启动，不跟随日志。
./start_offline.sh --detach 20260713

# 查询后台进程、YARN 任务和最近日志。
./status_offline.sh 20260713
```

## 4. Dashboard 前后端

Dashboard 前端已经打包到后端 Jar 的静态资源中，生产环境只需启动后端。

在 `slave3` 执行：

```bash
cd /home/master0/risk-dashboard
./start.sh
./status.sh
```

访问地址：`http://192.168.125.113:8080/`

停止：

```bash
cd /home/master0/risk-dashboard
./stop.sh
```

## 5. 实时风控链路

先启动 Dashboard，再启动实时 Spark 消费程序，最后启动数据产生端。数据产生端会将交易写入 MySQL 和 Kafka `trans-event`；实时程序从 Kafka 计算风险并调用 Dashboard 告警接口。

在 `master0` 执行：

```bash
cd /home/master0/risk-streaming
./start.sh
./status.sh
```

停止：

```bash
cd /home/master0/risk-streaming
./stop.sh
```

在 Windows PowerShell 中执行：

```powershell

Set-Location "$root\risk-dashboard-frontend"
& 'E:\nodejs\npm.cmd' install --no-audit --no-fund
& 'E:\nodejs\npm.cmd' run build

robocopy .\dist "$root\risk-dashboard-backend\src\main\resources\static" /MIR
if ($LASTEXITCODE -gt 7) { throw "Frontend asset copy failed: $LASTEXITCODE" }

Set-Location "$root\risk-dashboard-backend"
& 'E:\codex_data\tools\apache-maven-3.9.11\bin\mvn.cmd' `
  -s 'E:\codex_data\tools\maven-settings-aliyun.xml' `
  '-Dmaven.repo.local=E:\codex_data\tools\m2' `
  -DskipTests package
```

生成 Jar：

```text
risk-dashboard\risk-dashboard-backend\target\risk-dashboard-backend-1.0.0.jar
```

## 7. 最终测试顺序

1. 清空 MySQL、Redis、HDFS 和 Kafka 项目数据。
2. 启动基础服务。
3. 运行数据产生端，先生成历史数据到 MySQL。
4. 校验历史数据的 `trans_timestamp` 分布。
5. 启动离线分析，等待 ADS 数据写入 MySQL 和 Redis。
6. 启动 Dashboard，检查离线分析页面。
7. 启动实时风控程序。
8. 启动实时数据产生端，检查 Dashboard 的告警、地图与近 60 秒指标。
