# risk_control 数据库结构文档

## 1. 概览

- 共 25 张表，均采用 `InnoDB` 存储引擎和 `utf8mb4_general_ci` 排序规则。
- 表间关联通过业务标识字段实现。

| 表名 | 用途 |
|---|---|
| `ads_city_risk_rank` | 离线城市风险排行。 |
| `ads_counterparty_risk_rank` | 离线收款方风险排行。 |
| `ads_cross_region_risk_flow` | 离线跨区域风险流。 |
| `ads_device_risk_rank` | 离线设备风险排行。 |
| `ads_high_risk_transaction` | 离线高风险交易明细。 |
| `ads_high_risk_user_rank` | 离线高风险用户排行。 |
| `ads_offline_overview_metrics` | 离线分析总览指标。 |
| `ads_offline_score_distribution` | 离线评分桶分布。 |
| `ads_province_risk_rank` | 离线省份风险排行。 |
| `ads_risk_behavior_distribution` | 离线风险行为分布。 |
| `ads_risk_feature_distribution` | 离线风险特征分布。 |
| `ads_risk_time_trend` | 离线风险时间趋势。 |
| `ads_rule_risk_rank` | 离线规则命中排行。 |
| `ads_transaction_risk_detail` | 离线交易风险事实明细。 |
| `counterparty_blacklist` | 离线生成的收款方风险库。 |
| `history_risk_seed_log` | 历史风险种子记录。 |
| `login_event` | 登录事件数据。 |
| `risk_alert` | 实时规则命中后的告警与处置记录。 |
| `sys_user` | 风险大屏用户和角色。 |
| `trader_counterparty` | 用户与收款方关系。 |
| `trader_user` | 交易账户主数据。 |
| `transaction_history` | 原始交易流水；网关/模拟器写入，实时与离线链路读取。 |
| `trans_chain` | 实时引擎生成的资金链路节点。 |
| `t_metrics` | 分钟级实时指标历史快照。 |
| `user_profile` | 离线用户历史行为画像。 |

## 3. 逻辑关联

| 关联 | 字段 | 说明 |
|---|---|---|
| 交易与告警 | `transaction_history.trans_id` -> `risk_alert.trans_id` | 实时规则命中的交易生成告警。 |
| 交易与链路 | `transaction_history.trans_id` -> `trans_chain.trans_id` | 实时引擎写入资金链路节点。 |
| 用户与画像 | `trader_user.user_id` -> `user_profile.user_id` | 离线作业形成用户画像。 |
| 用户与登录 | `trader_user.user_id` -> `login_event.user_id` | 登录数据参与画像与规则。 |
| 交易与收款方风险 | `transaction_history.counterparty_id` -> `counterparty_blacklist.counterparty_id` | 实时收款方规则查询。 |
| 历史风险与交易 | `history_risk_seed_log.trans_id` -> `transaction_history.trans_id` | 离线风险种子关联交易。 |
| 离线快照 | 所有 `ads_*` 表按 `dt` | 离线分析日期快照。 |

## 4. 使用说明

- `risk_alert` 为 21 列，含处置字段 `status`、`handler`、`handle_time`、`handle_remark`。
- `ads_*` 表是离线 MapReduce 分析后写入 MySQL 的应用层结果；`t_metrics` 为 Redis 实时指标的分钟级历史快照。
- 毫秒级时间戳字段包括 `trans_timestamp`、`login_time`、`snapshot_time`、`event_time`、`registration_time`、`last_trans_ts`、`last_login_time`、`handle_time`。
